# IoC和DI
+ <font style="background-color:#FBDE28;">IoC</font>：Inversion of Control（控制反转）

控制：资源的控制权（资源的创建、获取、销毁等）

反转：和传统的方式不一样了

<font style="color:#DF2A3F;">IoC是一种</font><u><font style="color:#DF2A3F;">思想</font></u><font style="color:#DF2A3F;">，将资源的控制权都交给容器来做</font>

+ <font style="background-color:#FBDE28;">DI</font> ：Dependency Injection（依赖注入）

依赖：组件的依赖关系，如`NewsController`依赖`NewsServices`

注入：通过setter方法、构造器、等方式自动的注入（赋值）

<font style="color:#DF2A3F;">DI是</font><u><font style="color:#DF2A3F;">实现</font></u><font style="color:#DF2A3F;">，发现资源的需求，将其他容器的需求通过</font><u><font style="color:#DF2A3F;">setter方法、构造器</font></u><font style="color:#DF2A3F;">等方式自动的注入</font>

例如婚介所，将自己的信息，和需求注册到婚介所，成为会员。婚介所会根据不同人的信息和需求给你匹配上自己需要的，不在是自己在大街上主动找了。

### <font style="color:rgb(44, 44, 54);">获取组件</font>
精确获取容器中的组件对象；可以从组件的`<u><font style="color:#DF2A3F;">名称</font></u><u>和</u><u><font style="color:#DF2A3F;">类型</font></u>`获取

（这里的ioc是指`ApplicationContext`。spring应用上下文，也就是`IoC容器`）

1. 从<font style="background-color:#FBDE28;">名称</font>获取：

```java
Person zhangsan = (Person) ioc.getBean("zhangsan");
System.out.println("对象 = " + zhangsan);
```

这种如果多个组件名称相同，<u><font style="color:#DF2A3F;">一定只会给容器中放一个最先声明的哪个。</font></u>

如果组件不存在组件不存在，抛异常：`NoSuchBeanDefinitionException`

2. 从<font style="background-color:#FBDE28;">类型</font>获取
    - 获取一个

```java
//按照组件类型获取对象
Person bean = ioc.getBean(Person.class);
System.out.println("bean = " + bean);
```

如果该类型的组件有多个，类型只要一个：抛异常：NoUniqueBeanDefinitionException

    - 获取多个

```java
//按照组件类型获取这种类型的所有对象
Map<String, Person> type = ioc.getBeansOfType(Person.class);
System.out.println("type = " + type);
```

返回所有组件的集合（Map）。

3. 按照<font style="background-color:#FBDE28;">类型</font>和<font style="background-color:#FBDE28;">名称</font>

```java
//按照类型+名字
Person bean = ioc.getBean("zhangsan", Person.class);
System.out.println("bean = " + bean);
```

# 注册组件
## @Bean方式注册组件
```java
import org.springframework.context.annotation.Bean;
import org.springframework.context.annotation.Configuration;
@Configuration
public class AppConfig {
    @Bean(name = "mySpecialService")
    public MyService myService() {
        return new MyService();
    }
}
```

`myService`<font style="color:rgba(0, 0, 0, 0.85);">方法上使用了</font>`@Bean`<font style="color:rgba(0, 0, 0, 0.85);">注解。当 Spring 容器加载这个配置类时，会调用这个方法，将方法返回的</font>`MyService`<font style="color:rgba(0, 0, 0, 0.85);">对象作为一个 Bean 添加到容器中。</font><u><font style="color:#DF2A3F;">这个 Bean 的名称默认是方法名（在这里是</font></u>`<u><font style="color:#DF2A3F;">myService</font></u>`<u><font style="color:#DF2A3F;">），也可以通过</font></u>`<u><font style="color:#DF2A3F;">@Bean</font></u>`<u><font style="color:#DF2A3F;">注解的</font></u>`<u><font style="color:#DF2A3F;">name</font></u>`<u><font style="color:#DF2A3F;">属性来指定名称，例如</font></u>`<u><font style="color:#DF2A3F;">@Bean(name = "mySpecialService")</font></u>`<u><font style="color:#DF2A3F;">。</font></u>

<font style="color:rgb(44, 44, 54);">注解中指定一些属性，比如初始化和销毁方法等：</font>`<font style="color:rgb(44, 44, 54);">@Bean(initMethod = "init", destroyMethod = "cleanup")</font>`

## `@Scope`调整单例为多例
```java
@Scope 调整组件的作用域：
1、@Scope("prototype")：非单实例:
    1.容器启动的时候不会创建非单实例组件的对象。
    2.什么时候获取，什么时候才创建
2、@Scope("singleton")：单实例： 默认值
    1.容器启动的时候会创建单实例组件的对象。
    2.容器启动完成之前就会创建好
@Lazy：懒加载
    容器启动完成之前不会创建懒加载组件的对象
    什么时候获取，什么时候创建
3、@Scope("request")：同一个请求单实例
4、@Scope("session")：同一次会话单实例
```

单例，在容器启动之前就<font style="color:#DF2A3F;">提前创建好，默认就是这个</font>

<font style="color:#000000;">多例，启动之前不创建，</font><font style="color:#DF2A3F;">使用的时候再创建</font>

<font style="color:#DF2A3F;">@Lazy：懒加载</font>模式，容器使用之前不创建，配合`<font style="color:#F38F39;">@Scope("singleton")</font>`使用，因为本身是在使用之前就创建好，与之相反的是<u>饿汉模式</u>

<u>这两个注解都放在组件上</u>

## 工厂模式创建实例
场景：如果制造某些对象比较复杂的时候，利用工厂方法进行创建。

FactoryBean在<u>容器中放的组件的类型，是接口中泛型指定的类型</u>，组件的名字是 工厂自己的名字

```java
@Component
public class BYDFactory implements FactoryBean<Car> {


    /**
     * 调用此方法给容器中制造对象
     * @return
     * @throws Exception
     */
    @Override
    public Car getObject() throws Exception {
        System.out.println("BYDFactory 正在制造Car对象...");
        Car car = new Car();
        return car;
    }


    /**
     * 说明造的东西的类型
     * @return
     */
    @Override
    public Class<?> getObjectType() {
        return Car.class;
    }


    /**
     * 是单例？
     *    true：是单例的；
     *    false：不是单例的；
     * @return
     */
    @Override
    public boolean isSingleton() {
        return true;
    }
}
```

## 条件注册


首先组件注入上加入`<font style="color:#F38F39;">@Conditional(条件类名.class)</font>`，当条件满足时，才将这个组件注入到容器中

如在该注解在类上，那么整个类上的，整个类都不满足要求，类中的方法都没用

```java
@Configuration //告诉Spring容器，这是一个配置类
public class PersonConfig {

    //场景：判断当前电脑的操作系统是windows还是mac
    //  windows 系统，容器中有 bill
    //  mac 系统，容器中有 joseph
    @Conditional(MacCondition.class)
    @Bean("joseph")
    public Person joseph(){
        Person person = new Person();
        person.setName("乔布斯");
        person.setAge(20);
        person.setGender("男");
        return person;
    }

    @Conditional(WindowsCondition.class)
    @Bean("bill")
    public Person bill(){
        Person person = new Person();
        person.setName("比尔盖茨");
        person.setAge(20);
        person.setGender("男");
        return person;
    }
}
```

条件类必须实现`Condition`接口,返回false代表条件不匹配。返回true，代表条件匹配。

```java
public class MacCondition implements Condition {
    @Override
    public boolean matches(ConditionContext context, AnnotatedTypeMetadata metadata) {
        Environment environment = context.getEnvironment();

        String property = environment.getProperty("OS");

        return property.contains("mac");
    }
}
```

## `<font style="color:#F38F39;">@Qualifier</font>`和`<font style="color:#F38F39;">@Primary</font>`
### <font style="color:#F38F39;">@Qualifier</font>是可以精确指定名称
```java
@Qualifier("zhangsan")
@Autowired
Person person;
```

使用场景:

+ 新老版本交替,老版本并不支持变量名找容器名,需要用此注解指定.
+ 如果写Person写的都是person,后续想换,需要将代码中的所有person都进行修改,此时在注入的地方加上`@Qualifier`精确指定

### <font style="color:#F38F39;">@Primary</font>默认组件
如果存在类型一样重复的组件,我们可以在注册组件的地方加入`@Primary`,这样注入的时候,都会注入标了<font style="color:#F38F39;">@Primary</font>的组件.

```java
@Bena
@Primary("zhangsan")
public Person person(){
    Person person = new Person();
    person.setName("张三");
    return person;
}
```

```java
@Autowired
Person person;
```

这样再注入就不会报错,如果我们修改变量名,注入的也依然是指定了`primary`的组件,此时的<u>变量名只是修改了组件的名称</u>

## 构造器注入(spring推荐这种)
因为spring会自动去容器中找到构造器需要的所有参数组件.<u>利用</u>

<u>这个有参构造器,强制spring自动注入</u>

<u>这种构造器的好处就是,以后脱离的spring的环境,我们想用UserDao,只能new,但是只有一个有参构造,我们只能再new Dog,依赖关系已经固定了.如果使用的是注解自动注入,使用UserDao时,不确定需要哪些依赖,如果脱离spring环境,注解也就没用了,就会出大问题.</u>

```java
/**
* 例如我们在给UserDao类上加入@Resource注解
*/
@Repository
public class UserDao{
    Dog dog;
     //@Autowired // 如果有多个构造函数，则需要此注解来指定哪一个用于自动装配
    public UserDao(Dog dog){
        this.dog=dog;
    }
}
```

属性名的作用只是给这个组件起个名称，并不决定注入的是什么，注入的取决于构造方法的这个参数。

## <font style="color:rgb(44, 44, 54);">set方法注入</font>
Spring创建组件的时候,一个Java bean里有什么属性,并不取决于写了啥属性名(Dog dog),而是有没有个属性的get、set方法

```java
@Repository
public class UserDao{
    Dog dog;
    @Autowired
    public void setDog(@Qualifier("dog") Dog dog){
        this.dog = dog;
    }
}
```

属性名的作用只是给这个组件起个名称，并不决定注入的是什么，注入的取决于set方法的这个参数。

## xxxAware感知接口
Aware只是一个标记，ctrl+h查看实现，有很多xxxAware。

例如：如果有一个组件，想要使用环境变量，就需要有`Environment`这个对象，而我们implement实现一个`EnvironmentAware`，这个接口有一个set方法，这个`set`方法是有spring调用的，（前提类是容器中的组件）。spring为你创建对象的时候，发现你实现了`EnvironmentAware`感知接口，说明感兴趣`Environment`这个玩意，就会将`Environment`传进来。

```java
@Service
public class HahaService implements EnvironmentAware, BeanNameAware {

    private Environment environment;
    private String myName;

    @Override
    public void setEnvironment(Environment environment) {
        this.environment = environment;
    }
    
    public String getOsType(){
       return environment.getProperty("OS");
    }
    
    @Override
    public void setBeanName(String name) {
        this.myName = name;
    }
}
```

## `<font style="color:#F38F39;">@Value</font>`
我们让spring容器创建一个组件的时候，里面属性默认是空的，可以使用`@Value`注入一些基本属性，前面的注入都是注入组件。

1. `@<font style="color:#F38F39;">Value</font>("字面值")`，直接赋值

```java
@Component
public class Dog{
    @Value("旺财")
    private String name;
}
```

2. `@<font style="color:#F38F39;">Value</font>("${key}")`：从<font style="color:#DF2A3F;">application配置文件</font>中动态取出

还可以实现 如果去配件文件取不到设置默认值：`@<font style="color:#F38F39;">Value</font>("${key：xiaogou}")`

```java
@Component
public class Dog{
    @Value("${dog.name}")
    private String name;
}
```

```properties
dog.name = 旺财
```

3. `@<font style="color:#F38F39;">Value</font>("#{SpEL}")`：spring EXpression Language：spring 表达式

也就是可以在大括号（#{}）里使用表达式，静态调用要加一个`T`

```java
@Value("#{T(java.util.UUID).randomUUID().toString()}")
private String id

@Value(#{1>2?'haha':'hehe'})
private String flage
```

## `<font style="color:#F38F39;">@ProperSource</font>`
如果不是在application里写的数据，而是在其他properties等配置文件，需要用`<font style="color:#F38F39;">@ProperSource</font>`<font style="color:#F38F39;">指定。</font>

+ 说明属性来源，把指定的文件导入容器中，供我们使用，括号里面指定的路径使用类路径`classpath：conf/cat.properties`从自己项目的java类和resources目录下
+ 如果是引入的jar包，`classpath*:conf/cat.properties`

```java
@PropertySource("classpath:conf/cat.properties")
@Data
@Component
public class Cat {

    @Value("${cat.name:Tom}") // : 后面是取不到的时候的默认值；
    private String name;
    @Value("${cat.age:20}")
    private int age;

}
```

```java
cat.name=Tom2222
cat.age=2
```

## `@Profile`多环境
`<font style="color:rgb(44, 44, 54);">@Profile</font>`<font style="color:rgb(44, 44, 54);"> 是 Spring 框架中的一个注解，它用于根据激活的环境配置来有条件地注册 bean。通过使用 </font>`<font style="color:rgb(44, 44, 54);">@Profile</font>`<font style="color:rgb(44, 44, 54);"> 注解，你可以定义特定于某个或某些环境的 bean，并且只有在这些环境被激活时，这些 bean 才会被创建和注入到应用程序上下文中。</font>

### <font style="color:rgb(44, 44, 54);">指定bean</font>
    1. **<font style="color:rgb(44, 44, 54);">单个 Profile</font>**<font style="color:rgb(44, 44, 54);"> 如果你想指定一个 bean 只有在特定的 profile 下才可用，可以像下面这样使用 </font>`<font style="color:rgb(44, 44, 54);">@Profile</font>`<font style="color:rgb(44, 44, 54);"> 注解：</font>

```java
@Component
@Profile("dev")
public class DevDataSourceConfig {
    // 配置代码
}
```

    2. **<font style="color:rgb(44, 44, 54);">多个 Profiles</font>**<font style="color:rgb(44, 44, 54);"> 如果你有一个 bean 应该在多个 profiles 中都可用，你可以传递一个字符串数组给 </font>`<font style="color:rgb(44, 44, 54);">@Profile</font>`<font style="color:rgb(44, 44, 54);"> 注解：</font>

```java
@Component
@Profile({"dev", "test"})
public class DevTestDataSourceConfig {
    // 配置代码
}
```

    3. **<font style="color:rgb(44, 44, 54);">逻辑非（排除某个 Profile）</font>**<font style="color:rgb(44, 44, 54);"> 如果你想在除了某个特定的 profile 外的所有其他环境中都加载一个 bean，可以使用感叹号(!)前缀：</font>

```java
@Component
@Profile("!prod")
public class NonProdDataSourceConfig {
    // 配置代码
}
```

以上这些还可以标在类上

### <font style="color:rgb(44, 44, 54);">激活 Profiles</font>
<font style="color:rgb(44, 44, 54);">你可以通过多种方式来激活 profiles，例如通过命令行参数、JVM 系统属性或者在 </font>`<font style="color:rgb(44, 44, 54);">application.properties</font>`<font style="color:rgb(44, 44, 54);"> 文件中设置，默认是</font>`<font style="color:rgb(44, 44, 54);">default</font>`<font style="color:rgb(44, 44, 54);">：</font>

+ <font style="color:rgb(44, 44, 54);">命令行：</font>`<font style="color:rgb(44, 44, 54);">--spring.profiles.active=dev</font>`
+ <font style="color:rgb(44, 44, 54);">JVM 参数：</font>`<font style="color:rgb(44, 44, 54);">-Dspring.profiles.active=dev</font>`
+ <font style="color:rgb(44, 44, 54);">properties 文件：</font>

```properties
spring.profiles.active=dev
```

## `InitializingBean`和`DisposableBean`
组件实现`InitializingBean`接口，重写`afterPropertiesSet()`方法，实现在属性设置之后进行调用，也就是set赋值完成了调用该方法

实现`DisposableBean`，重写`destroy()`方法，<font style="color:rgb(44, 44, 54);">容器在Bean调用销毁方法之前调用 </font>`<font style="color:rgb(44, 44, 54);">destroy()</font>`<font style="color:rgb(44, 44, 54);"> 方法。</font>

## `<font style="color:#F38F39;">@PostConstruct</font>`和`<font style="color:#F38F39;">@PreDestroy</font>`
### @PostConstruct
+ **作用**：当一个 bean 被实例化并且所有必要的属性已经被设置之后，Spring 容器会在调用任何业务方法之前自动调用被 `@PostConstruct` 注解的**方法**。
+ **特点**：
    - 只会被调用一次，在容器创建该 bean 实例时。
    - 方法不能带有参数。
    - 方法必须是非静态的（static）。
    - 如果方法抛出异常，则容器将认为 bean 初始化失败，并通常会抛出异常或停止应用上下文的启动。
+ **用途**：适合用来执行一些初始化逻辑，比如加载资源、初始化缓存等。

```java
import javax.annotation.PostConstruct;

public class MyBean {

    @PostConstruct
    public void init() {
        // 执行初始化逻辑
    }
}
```

### <font style="color:rgb(44, 44, 54);">@PreDestroy</font>
+ **<font style="color:rgb(44, 44, 54);">作用</font>**<font style="color:rgb(44, 44, 54);">：在 Spring 容器销毁 bean 之前，它会自动调用被</font><font style="color:rgb(44, 44, 54);"> </font>`<font style="color:rgb(44, 44, 54);">@PreDestroy</font>`<font style="color:rgb(44, 44, 54);"> </font><font style="color:rgb(44, 44, 54);">注解的方法。</font>
+ **<font style="color:rgb(44, 44, 54);">特点</font>**<font style="color:rgb(44, 44, 54);">：</font>
    - <font style="color:rgb(44, 44, 54);">同样只调用一次，在容器准备移除该 bean 实例时。</font>
    - <font style="color:rgb(44, 44, 54);">方法不能带有参数。</font>
    - <font style="color:rgb(44, 44, 54);">方法必须是非静态的（static）。</font>
    - <font style="color:rgb(44, 44, 54);">如果方法抛出异常，容器仍然会继续销毁过程，但可能会记录错误日志。</font>
+ **<font style="color:rgb(44, 44, 54);">用途</font>**<font style="color:rgb(44, 44, 54);">：适合用来执行清理工作，例如释放资源、关闭连接等。</font>

```java
import javax.annotation.PreDestroy;

public class MyBean {

    @PreDestroy
    public void cleanup() {
        // 执行清理逻辑
    }
}
```

## `BeanPostProcessor`Bean外挂修改器
`<font style="color:rgb(44, 44, 54);">BeanPostProcessor</font>`<font style="color:rgb(44, 44, 54);"> 是 Spring 框架中一个非常强大的接口，它允许开发者在 Spring 容器实例化 bean 的前后对 bean 进行自定义处理。通过实现 </font>`<font style="color:rgb(44, 44, 54);">BeanPostProcessor</font>`<font style="color:rgb(44, 44, 54);"> 接口，你可以定义两个回调方法：</font>`<font style="color:rgb(44, 44, 54);">postProcessBeforeInitialization</font>`<font style="color:rgb(44, 44, 54);"> 和 </font>`<font style="color:rgb(44, 44, 54);">postProcessAfterInitialization</font>`<font style="color:rgb(44, 44, 54);">。这两个方法会在容器调用初始化回调（如 </font>`<font style="color:rgb(44, 44, 54);">@PostConstruct</font>`<font style="color:rgb(44, 44, 54);"> 或 </font>`<font style="color:rgb(44, 44, 54);">InitializingBean.afterPropertiesSet()</font>`<font style="color:rgb(44, 44, 54);">）之前和之后被调用。</font>

### <font style="color:rgb(44, 44, 54);">BeanPostProcessor 的用途</font>
+ **<font style="color:rgb(44, 44, 54);">增强或修改 bean 的行为</font>**<font style="color:rgb(44, 44, 54);">：可以在 bean 初始化前后对其进行增强，例如添加代理、设置额外属性等。</font>
+ **<font style="color:rgb(44, 44, 54);">验证配置</font>**<font style="color:rgb(44, 44, 54);">：检查 bean 配置是否正确，或者根据某些条件调整 bean 的属性。</font>
+ **<font style="color:rgb(44, 44, 54);">依赖注入</font>**<font style="color:rgb(44, 44, 54);">：除了标准的依赖注入方式外，还可以使用</font><font style="color:rgb(44, 44, 54);"> </font>`<font style="color:rgb(44, 44, 54);">BeanPostProcessor</font>`<font style="color:rgb(44, 44, 54);"> </font><font style="color:rgb(44, 44, 54);">来执行更复杂的依赖注入逻辑。</font>
+ **<font style="color:rgb(44, 44, 54);">拦截器</font>**<font style="color:rgb(44, 44, 54);">：可以用来作为拦截器，在 bean 方法调用前或后执行特定逻辑。</font>
